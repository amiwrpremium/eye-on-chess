# Invites API

Invite-code system for gated registration. New users must register with a valid invite code.

## REST Endpoints

### `GET /api/v1/invites/validate/:code` (Public)

Validate an invite code. Does not require authentication.

**Response (200):**

```json
{ "valid": true }
```

**Errors:** `404` invite not found, `410` invite already used.

### `GET /api/v1/invites/stats` (Auth Required)

Get the authenticated user's invite quota and usage.

**Response (200):**

```json
{
  "totalCreated": 10,
  "totalUsed": 8,
  "maxAllowed": 20,
  "remaining": 10,
  "canCreate": true,
  "usedTowardNext": 1,
  "neededForNext": 7
}
```

| Field            | Description                                            |
| ---------------- | ------------------------------------------------------ |
| `totalCreated`   | Total invites generated by user                        |
| `totalUsed`      | How many of those invites have been redeemed           |
| `maxAllowed`     | Current maximum invites the user can generate          |
| `remaining`      | Invites left to generate (`maxAllowed - totalCreated`) |
| `canCreate`      | Whether the user can generate more invites             |
| `usedTowardNext` | Progress toward unlocking the next batch               |
| `neededForNext`  | Used invites needed to unlock the next batch (7)       |

### `GET /api/v1/invites` (Auth Required)

List the authenticated user's invites with usage details. Ordered by creation date descending.

**Response (200):**

```json
{
  "invites": [
    {
      "id": "clx...",
      "code": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "used": true,
      "usedBy": "alice",
      "usedAt": "2026-03-20T12:00:00.000Z",
      "createdAt": "2026-03-19T08:00:00.000Z"
    }
  ]
}
```

### `POST /api/v1/invites` (Auth Required)

Generate a new invite code. Subject to quota enforcement.

**Response (200):**

```json
{ "code": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" }
```

**Errors:** `403` invite limit reached (response includes how many more used invites are needed to unlock the next batch).

## Quota System

- Each user starts with a batch of **10** invite codes.
- When **75%** (7 out of 10) of a batch's invites have been used by new registrants, the next batch of 10 is unlocked.
- Formula: `batchesEarned = 1 + floor(totalUsed / 7)`, `maxAllowed = batchesEarned * 10`.

## Registration

`POST /api/v1/auth/register` requires an `inviteCode` field in the request body. The invite must be valid (exists and unused). Upon successful registration the invite is marked as used.
